CiteBridge

One-button bidirectional sync between Zotero and Google NotebookLM. This guide takes you from a blank machine to a working sync — fast path first, full walkthrough and troubleshooting below.

⚠️ Read this first

The three things that trip up almost everyone.

  1. You need Python 3.10 or newer. Older versions can't install the NotebookLM library. Check with python3 --version.
  2. NotebookLM login needs a browser engine (Playwright + Chromium). This is the #1 reason setup stalls. The setup script installs it for you; if you install manually, run playwright install chromium before notebooklm login.
  3. Your NotebookLM login expires every few weeks. That's normal — it uses your Google session cookies. When sync suddenly fails with an "authentication expired" message, just re-run notebooklm login. Nothing is broken.
Why is NotebookLM finicky? Google has no public NotebookLM API for personal accounts. CiteBridge drives NotebookLM the same way your browser does, so it depends on your Google session staying valid. Your Zotero side uses an official, stable API and rarely needs attention.

For confident users

Fast path · ≈15 minutes

Comfortable with a terminal? This is the whole thing. Everyone else: skip to the walkthrough below.

# 1. Get the code
git clone https://github.com/RobVoss-AI/Citebridge.git
cd Citebridge

# 2. One-command setup (venv + deps + browser + login)
chmod +x setup.sh && ./setup.sh      # macOS / Linux
#   setup.bat                        # Windows: double-click instead

# 3. Launch the app
source .venv/bin/activate             # .venv\Scripts\activate on Windows
streamlit run app.py

Then, in the browser tab that opens (usually http://localhost:8501):

  1. Sidebar → Zotero Connection → paste API Key + Library ID → Save & Test Zotero.
  2. Sidebar → NotebookLM Connection → Verify NotebookLM Connection.
  3. Main page → Push to NotebookLM → check a collection → PUSH TO NOTEBOOKLM.

2

Set up the environment

Option A — automatic (recommended). Creates an isolated environment, installs everything (including the browser engine), and walks you through login:

chmod +x setup.sh && ./setup.sh     # macOS / Linux

On Windows, double-click setup.bat instead.

Option B — manual. Run each step yourself:

python3 -m venv .venv               # isolated environment
source .venv/bin/activate           # .venv\Scripts\activate on Windows
pip install -r requirements.txt     # CiteBridge dependencies
playwright install chromium         # one-time browser download
What's a virtual environment (.venv)? A sandbox that keeps CiteBridge's dependencies separate from your other Python tools. You activate it each time you open a new terminal — your prompt shows (.venv) when it's on.
3

Authenticate NotebookLM

notebooklm login

A Chromium window opens. Sign in to Google, wait for NotebookLM to load, then return to the terminal and press Enter to save your session. It's stored locally at ~/.notebooklm/storage_state.json and never leaves your machine.

See Playwright not installed? You skipped the browser step — run pip install "notebooklm-py[browser]" then playwright install chromium, and retry.

4

Get your Zotero API key + Library ID

  1. Go to zotero.org/settings/keys → Create new private key.
  2. Name it CiteBridge; check Allow library access and Allow write access (write enables note sync-back).
  3. Save Key and copy it — you won't see it again.
  4. Note your Library ID: the number near the top ("Your userID … is XXXXXXX"). It's a number, not your username.
5

Launch the app

source .venv/bin/activate     # .venv\Scripts\activate on Windows
streamlit run app.py

A tab opens at http://localhost:8501. Keep the terminal running while you use CiteBridge — closing it stops the app.

6

Connect Zotero

In the sidebar, expand 🔶 Zotero Connection: paste your API Key and Library ID, leave Library Type as user, optionally set your Zotero/storage path for faster PDF access, then click 💾 Save & Test Zotero for a green check.

7

Verify NotebookLM

Expand 🟣 NotebookLM Connection → 🔗 Verify NotebookLM Connection. This runs a real check, so the dashboard status is trustworthy:

StatusMeaningWhat to do
✅ ConnectedLive call succeeded this session.Start syncing.
🔑 Verify neededCredentials exist but unconfirmed.Click Verify.
❌ Not authenticatedNo login found.Run notebooklm login.
If Verify fails after you logged in earlier, your session expired — re-run notebooklm login and verify again.
8

Your first push (Zotero → NotebookLM)

Open the 📤 Push to NotebookLM tab, check one or more collections, and click PUSH TO NOTEBOOKLM. CiteBridge creates a notebook per collection and uploads its PDFs as sources. NotebookLM needs a little time to process each PDF; larger files take longer. Already-synced items are skipped on future runs.

Mental model

How it works — and why your data is safe

Zotero Collection Google NotebookLM ┌─────────────────┐ ┌──────────────────┐ │ AI Ethics │──push─►│ AI Ethics │ │ ├── paper1.pdf │ │ ├── paper1 (src) │ │ ├── paper2.pdf │◄─import┤ ├── paper2 (src) │ │ └── paper3.pdf │◄─notes─┤ └── AI Notes │ └─────────────────┘ └──────────────────┘

State tracking. A small local database at ~/.citebridge/sync_state.db records what's already synced (with file hashes), so repeat runs only handle new or changed items. Settings live at ~/.citebridge/config.yaml.

Data safety. CiteBridge only reads from Zotero, except when you opt in to writing notes back — and even then it only adds notes. It never deletes or modifies your existing items. Credentials never leave your machine. If the NotebookLM connection breaks, your Zotero library is untouched.

When something goes wrong

Troubleshooting

These are the actual messages you'll see, with the fix.

Authentication expired or invalid. Redirected to accounts.google.com…

Your NotebookLM session expired (every few weeks). Re-authenticate, then click Verify in the app:

source .venv/bin/activate
notebooklm login

Dashboard stuck on 🔑 "Verify needed"

Click Verify NotebookLM Connection in the sidebar. If it then errors, your session expired — re-run notebooklm login.

Playwright not installed. Run: pip install notebooklm[browser] …

You skipped the browser engine. Fix, then retry notebooklm login:

pip install "notebooklm-py[browser]"
playwright install chromium
notebooklm: command not found

Your virtual environment isn't active. Run source .venv/bin/activate (macOS/Linux) or .venv\Scripts\activate (Windows), then try again.

ModuleNotFoundError: No module named 'streamlit'

Same cause — the venv isn't active, or dependencies weren't installed. Activate the venv and run pip install -r requirements.txt.

NotebookLM library won't install / odd errors

You're likely on Python older than 3.10. Check python3 --version, install a newer Python, and recreate the venv.

Zotero connection failed

Re-copy the API key; confirm the Library ID is a number (not your username); ensure the key has library access enabled (and write access for note sync-back).

"No PDFs found" · slow sync · port in use

Some items simply lack a PDF. Set your Zotero/storage path for faster/reliable PDF access. Slow sync is normal — NotebookLM processes each PDF before continuing. Port busy? streamlit run app.py --server.port 8502.

Don't blindly upgrade the NotebookLM library. CiteBridge pins notebooklm-py to a tested version; newer releases change the API and can break imports. Avoid pip install --upgrade notebooklm-py unless you intend to re-test.
