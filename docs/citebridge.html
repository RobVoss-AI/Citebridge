Get it running

Deploy it locally in about 15 minutes

CiteBridge runs on your own computer. Here's the whole thing — the full walkthrough and a troubleshooting section live in the setup guide.

Python 3.10+
A free download from python.org.
Zotero
The desktop app, with a collection or two.
Google account
With access to NotebookLM.
~15 minutes
One-time setup, then it just runs.
Terminal — CiteBridge setup
# 1 · Get the code $ git clone https://github.com/RobVoss-AI/Citebridge.git $ cd Citebridge # 2 · One command installs everything & walks you through Google sign-in $ ./setup.sh # macOS / Linux · double-click setup.bat on Windows ✓ environment ready · browser installed · NotebookLM login complete # 3 · Launch the app $ streamlit run app.py ✓ open http://localhost:8501 in your browser
1

Get the code

Clone the repo (or download the ZIP from GitHub) and open the folder in a terminal.

2

Run the setup script

./setup.sh (or setup.bat on Windows) installs everything — including the browser engine — then opens Google sign-in for NotebookLM.

3

Get your Zotero API key

Create a free key at zotero.org/settings/keys (allow library + write access) and note your Library ID. Paste both into the app's sidebar.

4

Connect & sync

Launch with streamlit run app.py, verify both connections in the sidebar, pick your collections, and click Push.

Privacy by design

Why CiteBridge runs on your computer

CiteBridge is a personal tool, not a shared cloud service — and that's deliberate:

  • Your accounts stay yours. It uses your NotebookLM sign-in and your Zotero key, on your machine. Credentials never travel to anyone else's server.
  • Your library is never at risk. CiteBridge only reads from Zotero unless you turn on note sync-back — and even then it only adds notes.
  • The NotebookLM link refreshes periodically. Google sign-ins expire every few weeks; a quick re-login (about 30 seconds) restores it. We cover exactly how in the guide.
